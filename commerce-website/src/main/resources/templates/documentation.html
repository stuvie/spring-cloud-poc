about this website

This suite of 9 applications were all built on spring cloud

  1. config-server | spring-cloud-config provides centralized configuration for all other components
  2. registration and discovery server | eureka provides a service registry, to support fault tolerance and load balancing
  3. tracing server | zipkin provides a way to trace and analyze cascaded web service requests
  4. hystrix-dashboard | allows us to monitor the state of our hystrix circuit breakers
  5. commerce-website is the user-facing public website
  6. promotions-service | a microservice for in-store promotions, using H2 database
  7. promotions-service-v2 | a microservice for in-store promotions, using H2 database
  8. catalog-service | a microservice for the items we sell, using H2 database
  9. slow-downstream-service | a slow microservice to illustrate distributed tracing of slow requests

Features to Demonstrate

  • spring actuator - healthchecks and much more
  • distributed tracing of cascaded service calls with zipkin
  • automatic registration of services as they come online, via eureka service discovery
  • load balancing between 2 versions of promotions-service
  • hystrix circuit-breaker fallback when the promotions-service is down
  • website configuration refresh (edit config/website.properties, then click 'Refresh website config' button on admin portal)

Technologies used

  • spring-cloud-config server
  • eureka service registry
  • zipkin
  • hystrix circuit breaker - to provide fallback behavior if a service is down
  • zuul proxy - to proxy requests from the website to downstream services
  • spring load balancing between the 2 versions of promotions-service
  • spring actuator - used in every application to support healthcheck and monitoring
  • spring-jpa, spring-data-rest, spring-mvc, spring-boot, H2 database, etc.
